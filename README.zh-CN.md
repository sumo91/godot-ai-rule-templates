# Godot AI Rule Templates

适用于 Godot 游戏开发的 AI 规则模板，可复用于 Claude、Cursor 以及其他编程助手。

这个仓库的定位很简单：以后你新建一个 Godot 项目时，直接把合适的规则文件复制进去，按项目稍微改一下，就能马上开始稳定协作，而不是每次都重新写一套 AI 约束。

[English README](./README.md)

## 这个仓库能帮你解决什么

- 提供可以直接复制到新项目里的规则文件
- 明确哪些工作适合 AI 直接改文本，哪些更适合人类在 Godot 编辑器里完成
- 让不同工具之间的工作方式尽量保持一致
- 保持模板足够小，方便你以后按项目继续定制

## 文件怎么选

| 文件 | 适合什么情况 |
| --- | --- |
| `CLAUDE.md` | 你的 AI 助手会把仓库里的 Markdown 当作高优先级操作规则 |
| `AGENTS.md` | 你希望有更明确的多轮协作规则、交接要求和 session 纪律 |
| `.cursorrules` | 你的编辑器更适合读取短规则文件，而不是长篇 Markdown 文档 |

如果一个项目会同时使用多个工具，也可以把多份规则一起放在仓库根目录，只要它们的内容保持一致即可。

## 双语文件说明

- 英文规则文件通常更适合作为项目里真正给工具读取的主文件。
- 中文副本更适合你自己阅读、理解、讨论和二次定制。
- 不要默认所有 AI 工具都会自动发现或优先读取中文副本。
- 如果你修改了某条规则，最好同步更新对应的英文文件和中文副本。

## 中文对照文件

- `AGENTS.zh-CN.md`：`AGENTS.md` 的中文对照版

## 最快上手方式

1. 打开新的 Godot 项目根目录。
2. 复制你要用的规则文件进去：
   - `CLAUDE.md`
   - `AGENTS.md`
   - `.cursorrules`
3. 把它们放在仓库根目录，方便工具自动发现。
4. 按下面的清单做项目级调整。
5. 尽早提交这些规则，这样之后每次 AI 协作都会从同一套基线开始。

## 推荐默认组合

如果是你自己的单人 Godot 项目，而且可能会混用多个 AI 工具，一个实用的默认组合是：

- 保留 `CLAUDE.md`，作为更强的 agent 规则
- 保留 `.cursorrules`，作为编辑器层面的轻量约束
- 只有当你确实需要更明确的交接和 session 协议时，再加入 `AGENTS.md`

## 首次使用前建议修改

在把这些模板当成正式项目规则之前，建议至少检查这些内容：

- 是否要固定 Godot 版本
- 项目的目录约定，例如 `scenes/`、`scripts/`、`data/`、`addons/`、`tests/`
- 是否要补充团队自己的测试、提交、分支或评审规则
- 是否要加强对 scene inheritance、data-driven content、工具使用边界的约束
- 哪些场景必须改为人工接手，不能交给 AI 自动化处理

## 这些模板的设计原则

- 避免写入机器相关路径
- 避免写入项目私有假设
- 优先鼓励可 review、可 diff、可复现的文本工作流
- 明确文本编辑和 Godot 编辑器操作之间的边界
- 重点关注 Godot 工具选择、编码习惯和人机协作方式

## 复用或公开前再检查一次

如果你准备把这些模板复制到别的仓库，或者分享给团队使用，建议再确认：

- 当前文案是否真的匹配你实际在用的工具
- 有没有和你真实工作流冲突的规则
- `CLAUDE.md`、`AGENTS.md`、`.cursorrules` 之间是否保持同步
- 项目自己的 README 里是否写清楚了哪份规则文件最权威
